# Code of Conduct

## Our pledge

This project is intended to help people handle security-sensitive logs safely. Contributors, users, maintainers, and issue participants are expected to keep discussions respectful, practical, and focused on improving the tool.

## Expected behavior

- Be respectful and patient with people of different experience levels.
- Assume good faith when someone reports confusing behavior or a possible safety problem.
- Keep security discussions concrete and reproducible.
- Avoid posting real secrets, client data, raw logs, token maps, salts, or other private artifacts in public issues, pull requests, comments, screenshots, or examples.
- Use synthetic, scrubbed, or clearly fictional samples when discussing detections.

## Unacceptable behavior

- Harassment, personal attacks, insults, or repeated disruptive comments.
- Publishing someone else's private information.
- Posting raw sensitive logs or credentials.
- Using the project issue tracker to request help evading detection, hiding malicious activity, or exposing private data.
- Pressuring maintainers or contributors to review private logs outside an approved secure process.

## Reporting concerns

For conduct concerns, open a private security/contact path if available or contact the maintainer directly. For security-sensitive tool behavior, follow the security policy instead of posting raw data publicly.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block participants; or take other reasonable action to protect the project and its users.
